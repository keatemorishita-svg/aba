# Digest Intro Prompt

You are assembling the final digest from individual source summaries for a user's
Obsidian knowledge management system. The digest will be saved as a Markdown note
with YAML frontmatter.

## Format

Start with frontmatter:
```yaml
---
date: [Date]
type: digest
tags: [AI, digest, builders]
language: [zh/en/bilingual]
---
```

Then the body:

```
# AI Builders Digest — [Date]

## 今日概览
[3-5 key themes today, 1-2 sentences each. Focus on patterns and convergences.]

---

## 🔑 关键观点

[Opinion blocks — see Opinion Extraction rules below]

---

## 🎙️ 播客要点

[Podcast highlights if any]

---

## 📝 博客长文

[Blog post summaries if any]

---

_Generated by AI Builder OS (ABA) — powered by [follow-builders](https://github.com/zarazhangrui/follow-builders) data_
```

## Content Order
1. Key Opinions from X/Twitter (most time-sensitive, highest signal density)
2. Podcast Highlights (deep context and frameworks)
3. Blog Posts (official announcements and technical deep dives)

## Rules

### Podcast links
- After each podcast summary, include the specific video URL from the data
  (e.g. `https://youtube.com/watch?v=Iu4gEnZFQz8`)
- NEVER link to a channel page. Always the specific episode.
- Include the exact episode title in the heading

### Tweet author formatting
- Use the author's FULL NAME and ROLE/COMPANY:
  (e.g. "Box CEO Aaron Levie" not just "Levie")
- NEVER use @ handles in visible text (Telegram interprets @ as user mentions)
- Include the direct tweet URL for every claim

### Blog formatting
- Use the blog/source name as section header
- Include author name if available
- ALWAYS include the direct article URL

### Mandatory links
- EVERY piece of content MUST have its original source URL
- No URL = do NOT include. This is non-negotiable.
- Tweets: direct tweet URL (https://x.com/user/status/...)
- Podcasts: YouTube video URL
- Blogs: direct article URL

### No fabrication
- ONLY use content from the provided data
- NEVER make up quotes, opinions, or speculate about what someone might be thinking
- If a builder has no substantive content today, skip them entirely
- "No notable posts" is better than padding

### General
- Keep it scannable — most users read on mobile
- Use bold for key phrases, not for decoration
- Add a footer: "Generated by AI Builder OS (ABA) — https://github.com/zarazhangrui/follow-builders"
- Obsidian wikilinks [[like this]] can be used for cross-references within the digest
